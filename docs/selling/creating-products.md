# Creating Products

Transform your activities into products that can be sold in the ClaryNext marketplace. This guide covers product creation, configuration, and optimization.

## Product Types

### Single Activity

One activity as a product:
- User purchases, gets lifetime access
- Most straightforward option
- Good for standalone content

### Activity Bundle

Multiple activities together:
- Related activities packaged
- Discounted vs. individual purchase
- Users get all activities

### Series/Course

Sequential activities:
- Progressive curriculum
- Prerequisites enforced
- Certificate on completion

### Service Package

Activities plus services:
- Mentoring sessions included
- Custom elements
- Higher value offering

## Creating a Product

### From an Existing Activity

1. Open your activity in the designer
2. Click **Publish** or **Create Product**
3. Choose product type
4. Configure product settings
5. Submit for review/publish

### New Product from Scratch

1. Go to **Commerce > Products**
2. Click **New Product**
3. Select product type
4. Create or select activities
5. Configure and publish

## Product Configuration

### Basic Information

**Product Title**
- Clear, descriptive name
- Include key benefit
- Under 60 characters recommended

**Short Description**
- One-line summary
- Appears in listings
- Focus on outcome

**Full Description**
- Detailed product information
- What's included
- Who it's for
- What they'll achieve

### Media

**Thumbnail Image**
- Primary listing image
- Recommended: 800x600px
- Eye-catching and relevant

**Gallery Images**
- Additional preview images
- Screenshots of activity
- Show what users experience

**Preview Video**
- Optional but recommended
- 1-3 minute overview
- Show the value

### Categorization

**Category**
- Primary category assignment
- Determines where listed

**Tags**
- Searchable keywords
- Add relevant terms
- Improve discoverability

### Content

**What's Included**
- List of activities/components
- Estimated duration
- Features and benefits

**Prerequisites**
- Required prior knowledge
- Previous activities needed
- Technical requirements

**Learning Outcomes**
- What users will achieve
- Skills they'll gain
- Results they can expect

## Writing Effective Descriptions

### Product Title Best Practices

**Good Examples:**
- "Confident Communication: The DEAR MAN Method"
- "Weekly Goal Review: 15-Minute Clarity Session"
- "New Manager Toolkit: First 90 Days"

**Avoid:**
- Generic titles ("Communication Activity")
- Overly long titles
- Unclear value proposition

### Description Framework

**Hook** — Grab attention
```
Struggling to set boundaries without damaging relationships?
```

**Problem** — Acknowledge the challenge
```
Many of us know we should set boundaries but don't know
how to do it without conflict.
```

**Solution** — Your product
```
This activity teaches you the proven DEAR MAN technique
used by therapists worldwide...
```

**Benefits** — What they'll gain
```
After completing this activity, you'll:
- Know exactly what to say in boundary situations
- Feel confident having difficult conversations
- Maintain relationships while protecting your needs
```

**Call to Action** — Next step
```
Start now and have your first successful boundary
conversation this week.
```

## Pricing Your Product

### Pricing Strategy

Consider:
- Value delivered to user
- Comparable products
- Your expertise level
- Target audience budget
- Your revenue goals

### Price Points

Common ranges:
- Simple activities: $10-25
- Comprehensive activities: $25-50
- Bundles/courses: $50-150
- Premium programs: $150+

### Free Products

Reasons to offer free:
- Lead generation
- Build reputation
- Sample your style
- Community contribution

### Discounts

Set up promotional pricing:
- Launch discounts
- Bundle savings
- Seasonal sales
- Coupon codes

## Product Preview

### What Users Can Preview

Configure what's visible before purchase:
- Full description always visible
- Optional: First few steps
- Optional: Table of contents
- Sample content/screenshots

### Preview Best Practices

- Show enough to demonstrate value
- Don't give away the full product
- Build curiosity
- Demonstrate quality

## Product Review Process

### Submission

When you publish:
1. Product enters review queue
2. ClaryNext team reviews
3. Feedback provided if issues
4. Approved products go live

### Review Criteria

Products are checked for:
- Content quality
- Policy compliance
- Technical functionality
- Accurate descriptions
- Appropriate pricing

### Timeline

Typical review time:
- 1-3 business days
- Longer during high volume
- Expedited for established vendors

### If Rejected

If your product needs changes:
1. Review the feedback
2. Make required changes
3. Resubmit for review
4. Usually faster second review

## Managing Products

### Product Status

**Draft**
- Work in progress
- Not visible to public
- Can edit freely

**In Review**
- Submitted for approval
- Awaiting review
- Limited edits

**Active**
- Live in marketplace
- Can be purchased
- Updates possible

**Paused**
- Temporarily unavailable
- Existing customers keep access
- Can reactivate anytime

**Archived**
- Removed from marketplace
- Existing customers keep access
- Can be restored

### Updating Products

Make changes to live products:
1. Open product in Commerce
2. Edit as needed
3. Save changes
4. Updates apply immediately

:::note
Significant changes may trigger re-review.
:::

## Bundles and Series

### Creating a Bundle

1. Go to **Commerce > Products**
2. Click **New Bundle**
3. Add activities to include
4. Set bundle pricing
5. Configure bundle details
6. Publish

### Bundle Pricing

Typically discount vs. individual:
- 20-40% off individual total
- Makes bundle attractive
- Increases average order value

### Creating a Series

1. Create activities in sequence
2. Set prerequisites
3. Create series product
4. Configure completion certificate
5. Publish

### Series Benefits

- Progressive learning
- Higher completion rates
- Premium pricing justified
- Clear learning path

## Optimization Tips

### Increase Conversions

- **Strong title** — Immediate value clear
- **Quality images** — Professional appearance
- **Compelling description** — Focus on benefits
- **Social proof** — Reviews and testimonials
- **Appropriate price** — Value-aligned

### Gather Reviews

- Ask satisfied users to review
- Respond to all reviews
- Use feedback to improve
- Showcase reviews prominently

### Iterate Based on Data

- Review analytics regularly
- Test different descriptions
- Adjust pricing if needed
- Update based on feedback
